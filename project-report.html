1. Introduction

EduAI Smart School is a full-stack web application developed to establish a unified digital ecosystem connecting students, teachers, and parents within a single academic platform.

The system integrates classroom communication—including study notes dissemination, academic progress logging, and homework management—with artificial intelligence-based learning modules comprising a doubt resolution engine, automated study planner, adaptive quiz generator, and career counselling assistant. Additional engagement mechanisms such as experience points, study streaks, focus timers, and achievement badges are incorporated to promote sustained academic participation.

The platform is configured for Indian school curricula (CBSE, ICSE, and State Board) and implements role-based access control for students, teachers, parents, and administrators. It is intended for deployment at Air Force School Kalaikunda and comparable educational institutions.

2. Features of the Project

2.1 Multi-Role Portal System

Role Primary Functions
Student Dashboard, AI learning tools, homework management, focus timer, analytics, and gamification.
Teacher Study note publication, student observation logging, and class roster management.
Parent Monitoring of linked student progress, teacher remarks, and shared academic materials.
Administrator Role synchronisation and system administration.

Authentication is facilitated through role-specific identifiers: students authenticate via Enrollment Number, teachers via Teacher ID, and parents via registered email address linked to the student account.

2.2 Student Dashboard

The student dashboard provides consolidated access to the following modules:

  • Experience points and daily study streak tracking
  • Daily schedule, pending assignments, and upcoming examinations with countdown indicators
  • Teacher remarks categorised as academic, behavioural, or attendance-related observations
  • Curriculum notes and file attachments published by teaching staff

2.3 AI-Powered Learning Tools

Module Description Implementation
AI Doubt Solver Step-by-step explanations for mathematics, science, and programming, aligned with Indian curriculum standards. NVIDIA API (Gemma model); streaming response delivery
AI Study Planner Automated study schedules based on syllabus coverage, subject difficulty, and examination calendar. Anthropic Claude
AI Quiz Engine Topic-specific multiple-choice assessments with configurable difficulty and explanatory feedback. Anthropic Claude
AI Career Guide Career pathway recommendations, entrance examination guidance (JEE, NEET, CLAT), and structured action plans. Anthropic Claude

2.4 Teacher Portal

  • Class-filtered student roster with subject mapping
  • Publication of study notes with textual content and optional file attachments
  • Student progress logs for academic performance, conduct, and attendance
  • Class and subject assignment (e.g., Grade 10 Mathematics, Grade 11 Physics)

2.5 Parent Portal

  • Account linkage to student profiles via enrollment number or registered parent email
  • Support for monitoring multiple linked student accounts
  • Access to student experience points, study streaks, teacher observations, and shared materials
  • Performance analytics for linked students

2.6 Study and Productivity Modules

Module Functionality
Homework Tracker Assignment due dates, priority classification, and status management (pending, completed, overdue).
Focus Timer Pomodoro-based study sessions with configurable intervals and subject association.
Study Planner Calendar-based scheduling for study, revision, practice, and break sessions.
Examination Tracker Syllabus documentation and examination date monitoring.

2.7 Gamification and Analytics

The platform incorporates experience points, consecutive study day tracking, achievement badges, and level progression to encourage regular academic engagement. The analytics module provides weekly focus duration charts, subject-wise performance visualisation, and quiz score trends, with role-specific views for students, teachers, and parents.

4. Lessons Learned

4.1 Technical Observations

  1. Role-based access control through Supabase Row Level Security is essential for maintaining data integrity across student, teacher, and parent accounts.
  2. Institutional authentication via enrollment numbers and teacher identifiers is more practicable than email-exclusive login systems in school environments.
  3. Selection of appropriate AI service providers for distinct task categories—conversational tutoring versus structured content generation—improves system reliability.
  4. Comprehensive database schema planning prior to development reduces subsequent migration requirements.
  5. Streaming response delivery significantly enhances perceived responsiveness in AI-assisted tutoring interfaces.

4.2 Project Management Observations

  1. Incremental development commencing with the student portal prior to teacher and parent modules facilitated manageable scope control.
  2. Incorporation of Indian curriculum context within AI prompt engineering improved output relevance and accuracy.
  3. Implementation of security policies at the database layer from project inception is critical for educational data protection.
  4. Early deployment to cloud infrastructure enabled demonstration and evaluation without local environment constraints.

4.3 Challenges Addressed

  1. Resolution of Row Level Security recursion through dedicated database migration procedures.
  2. Synchronisation of user role metadata between authentication records and profile database entries.
  3. Implementation of parent–student account linkage via enrollment number at registration and within account settings.
  4. Application of class-scoped content filtering for study notes and progress logs.

4.4 Scope for Future Development

  1. Development of a mobile application or Progressive Web Application for extended device accessibility.
  2. Offline access capability for downloaded study materials.
  3. Voice-based input for AI tutoring interactions.
  4. Automated notification delivery to parents upon publication of teacher observations.
  5. Integration with institutional ERP and attendance management systems.

5. Conclusion

EduAI Smart School constitutes a comprehensive smart-school platform that unifies AI-assisted learning, teacher–parent communication, and student engagement within a single web-based system. The project demonstrates the feasibility of developing production-grade educational technology through structured software engineering practices.

Developed utilising contemporary web technologies and configured for Indian educational institutions including Air Force School Kalaikunda, the platform illustrates the practical application of artificial intelligence in routine classroom operations, thereby enhancing accessibility, transparency, and academic engagement across all stakeholders.
